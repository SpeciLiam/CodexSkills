# Peer Review

Review the requested scope: $ARGUMENTS

Act as the non-editing peer. If this is implementation work, review for bugs, missing states, maintainability, safety, and test gaps. If this is product/design work, review for user flow, hierarchy, copy, responsiveness, and polish.

Rules:

- Do not rewrite the whole solution.
- Rank findings by severity.
- Include file paths and line numbers when available.
- Say clearly when no high-confidence issues were found.
- End with the smallest verification step the executor should run.
