Use this file as the high-context profile that supports resume tailoring.

The goal is to give more detail than the one-page resume so a tailored version can:

## Default formatting and style rules

- Treat this README and the generic resume as the high-context evidence source, not the final length target
- It is fine for the generic source material to be richer than a one-page tailored resume because tailored copies should selectively compress from this context
- Every tailored company-specific resume must render to exactly one page
- Every tailored company-specific resume should use the full page effectively; avoid leaving large unused gaps when stronger truthful content can fit
- Keep the resume bullet-based; do not convert experience into paragraph summaries
- Let a bullet wrap to two lines when the content needs that space, but do not pad or stretch bullets just to fill lines
- Avoid bullets that leave only one or two low-information words on the wrapped line; rewrite them so the second line carries a meaningful phrase
- When a bullet wraps awkwardly, prefer improving the wording or adding better information over leaving an orphaned trailing word such as `UX` or `flows`
- Prefer clear, direct phrasing over compressed wording that reads awkwardly
- Avoid blank-looking header rows; always keep the contact line compact and include the personal website
- Keep the skills section visually full; aim for four balanced categories rather than a thin or underfilled block
- When space is tight, cut lower-value or older content before weakening the strongest recent bullets
- Prefer `Software Engineer` over `Software Engineer Intern` in tailored resume copies
- Avoid wording like `achieved >90\%`; prefer natural phrasing such as `reaching over 90\%`
